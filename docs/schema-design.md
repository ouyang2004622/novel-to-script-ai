# YAML Schema 设计说明

## 设计目标

本项目的主要功能是帮助作者把小说内容改编并整理成结构化剧本。Schema 首先服务于小说到剧本的改编流程，同时保留场景、节拍、镜头和声音提示等字段，方便后续继续拓展 AI 视频生成、漫剧分镜和镜头细化能力。Schema 采用 `script -> chapter -> scene -> beat` 的三级叙事结构，原因是：

- `chapter` 保留小说原始章节边界，方便追溯来源。
- `scene` 表达可视化场景，适合分镜、场景切换和镜头组织。
- `beat` 表达单个动作、对白、旁白、情绪或转场，是后续生成镜头和视频片段的最小单位。

这种结构既能保留小说章节顺序，也能让作者先得到可阅读、可编辑的剧本草稿；在后续拓展中，也可以继续作为 AI 分镜和视频生成的输入基础。

## 顶层结构

顶层只有一个必填字段 `script`。这样可以把所有剧本内容包在统一命名空间中，后续如果需要增加导出信息或校验元数据，也不会影响剧本主体结构。

| 字段 | 类型 | 作用 | 设计原因 |
| --- | --- | --- | --- |
| `script.title` | string | 剧本标题 | 用于页面展示、文件命名和导出标题。 |
| `script.source_type` | string | 来源类型 | 当前固定为 `novel`，说明数据来自小说文本。 |
| `script.language` | string | 内容语言 | 便于后续生成提示词、字幕或多语言导出。 |
| `script.chapters` | array | 章节列表 | 保留小说章节结构，方便从原文追溯到剧本。 |

## Chapter 字段

`chapter` 对应小说中的一个章节，负责承接原文结构和章节摘要。

| 字段 | 类型 | 作用 | 设计原因 |
| --- | --- | --- | --- |
| `chapter_id` | string | 章节唯一标识 | 方便后续引用、排序和定位。 |
| `chapter_title` | string | 章节标题 | 保留原文标题或整理后的标题。 |
| `summary` | string | 章节摘要 | 帮助用户快速理解章节内容，也方便 AI 后续处理上下文。 |
| `scenes` | array | 场景列表 | 把章节拆成可视化场景，便于分镜和视频生成。 |

## Scene 字段

`scene` 是可视化组织单位，描述一组发生在同一地点或同一段连续情境中的剧情。

| 字段 | 类型 | 作用 | 设计原因 |
| --- | --- | --- | --- |
| `scene_id` | string | 场景唯一标识 | 方便场景排序、编辑和后续引用。 |
| `scene_title` | string | 场景标题 | 用于页面展示和分镜分组。 |
| `location` | string | 场景地点 | 视频生成需要明确空间信息。 |
| `time` | string | 场景时间 | 帮助确定光线、氛围和画面风格。 |
| `characters` | string[] | 出场角色 | 帮助生成角色一致性和对白归属。 |
| `scene_purpose` | string | 场景作用 | 说明本场戏推动剧情、制造冲突或展示情绪的目的。 |
| `beats` | array | 场景内节拍 | 将场景拆成更小的动作、对白和镜头提示。 |

## Beat 字段

`beat` 是后续 AI 漫剧分镜和视频生成最重要的基础单位。每个 beat 表达一个连续的小动作、对白、旁白、情绪变化或转场。

| 字段 | 类型 | 作用 | 设计原因 |
| --- | --- | --- | --- |
| `beat_id` | string | beat 唯一标识 | 方便逐条编辑、重排和追踪生成结果。 |
| `type` | enum | beat 类型 | 限定为 `action`、`dialogue`、`narration`、`emotion`、`transition`，避免输出发散。 |
| `character` | string | 关联角色 | 对白和情绪 beat 需要知道说话或表现情绪的人物。 |
| `content` | string | beat 内容 | 描述要生成的动作、台词、旁白或转场。 |
| `duration` | number | 建议时长 | 可选字段，用于后续视频节奏控制。 |
| `camera_hint` | string | 镜头提示 | 可选字段，用于提示近景、远景、推镜、特写等画面组织。 |
| `sound_hint` | string | 声音提示 | 可选字段，用于提示环境声、音乐、音效或语气。 |

## Beat 类型说明

- `action`：人物动作或可视化事件，例如“林晚拿起长剑进入舞蹈室”。
- `dialogue`：角色对白，需要尽量填写 `character`。
- `narration`：旁白或必要解释，适合补充故事背景。
- `emotion`：人物情绪变化，适合引导表情、姿态和画面氛围。
- `transition`：场景或镜头转场，例如“雨声渐弱，画面切到灯塔”。
